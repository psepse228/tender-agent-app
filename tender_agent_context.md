## 1. Что это за проект

**Tender Agent** — флагманский клиентский продукт Solura. Автоматизированная система мониторинга тендеров для финансово-консалтинговой / аудиторской компании в Ташкенте.

**Суть:** система автоматически скрапит тендерные площадки, AI оценивает каждый тендер на соответствие профилю компании-клиента, результаты отображаются в Telegram Mini App с удобным интерфейсом карточек.

## 5. Архитектура n8n — логика workflow

### Общий поток:

```
[Schedule Trigger]
       ↓
[Firecrawl — скрапинг каждой площадки]
       ↓
[Парсинг / нормализация данных]
       ↓
[Получение профиля компании из Airtable]
       ↓
[GPT-4o — оценка тендера против профиля]
       ↓
[Airtable Save — сохранение результата]
       ↓
[Возврат данных в Telegram Mini App]
```

### Ключевые технические детали:

- **AI-узел:** используется стандартный OpenAI "Message a model" (GPT-4o), **не** AI Agent node — он ненадёжен для выражений и сложных цепочек
- **Скоринг тендеров:** отсутствие бюджета или низкий бюджет трактуется как средний диапазон (40–50 баллов), а не ноль — иначе процент совпадения падает ниже порога и нормальные тендеры фильтруются
- **Workflow 2:** между узлами "Parse AI Results" и "Return JSON" должен быть добавлен узел **Airtable Save** (нужно подтвердить, что добавлен)

## 8. Паттерны разработки (важно знать)

- **Не перестраивать весь workflow JSON** без запроса — при изменениях описывать конкретный узел + поле + новое значение
- **Концепт перед кодом** — сначала подтвердить подход, потом писать
- **Полные JSON-файлы** для n8n workflow предпочтительнее пошагового кликания в браузере (импорт через UI)
- **AI-скоринг:** missing budget → 40–50, не 0
- При изменениях n8n: указывать `node name → field → new value`
